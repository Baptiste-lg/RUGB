RUGB

Game Boy Emulator — User Guide

Getting Started

RUGB is a Game Boy (DMG) emulator that runs entirely in your browser. No downloads, no plugins — just load a ROM and play.

  1. Open the emulator
  2. Click Load ROM in the side menu (click the ☰ button top-left), or drag & drop a .gb or .zip file anywhere on the page
  3. Play!
You need to provide your own ROM files. RUGB does not include any games.

Remapping

Open the side menu and click Remap Keys to change any binding. Click a button, then press the key you want. You can export and import your bindings as JSON.

Gamepad

Connect any standard controller — it's auto-detected. Open Remap Controller in the menu to customize bindings. The left stick works as a D-pad.

On-Screen Buttons

Click or tap the buttons on the Game Boy shell. They animate when pressed, whether by mouse, touch, or keyboard.

Features

Save States

5 persistent slots + quick save/load. Export and import saves as JSON files to share or backup.

Battery Save

Games with battery backup (Pokemon, Zelda, etc.) auto-save to your browser. Progress persists between sessions.

Speed Control

½x slow motion, 1x normal, 2x, 4x, and uncapped fast forward by holding Space.

Turbo Buttons

Press Q or W to toggle auto-repeat for A or B. Great for rapid-fire or text-heavy games.

Color Palettes

Classic green, gray, B&W, or define your own custom 4-color palette with color pickers.

Audio Channels

Mute individual channels (CH1–CH4) to isolate instruments or debug audio.

Screenshot

One click to download the current screen as a PNG image.

Fullscreen

Press F11 or use the menu to go fullscreen. Works in both console and screen-only modes.

Display Modes

Two display modes are available in the side menu:

  • Console — the full Game Boy shell with screen, buttons, and speaker grille. Resizable by dragging the edges.
  • Screen Only — just the game screen, no shell. Useful for larger displays or recording.

Save System

Battery Save (In-Game Saves)

Games with built-in save functionality (like Pokemon) store data in cartridge RAM. RUGB automatically saves this to your browser's localStorage every 5 seconds and when you close the tab. When you load the same ROM again, your save is restored.

Battery saves are tied to your browser. Clearing browser data will delete them. Use save state export for important saves.

Save States

Save states capture the entire emulator state — you can save and restore at any moment, not just at save points. Open Save States in the menu to manage 5 slots.

  • Save — capture current state to a slot
  • Load — restore from a slot
  • Export — download the save as a JSON file
  • Import — load a previously exported save into an empty slot

Quick Save / Load

Press F5 to quick save and F8 to quick load. This uses a separate in-memory slot (not persisted — lost when you close the tab).

Supported Games

RUGB supports original Game Boy (DMG) games using the following cartridge types:

No MBC (ROM only)Tetris, Dr. Mario, Alleyway
MBC1Super Mario Land, Kirby's Dream Land, Metroid II, Donkey Kong
MBC3Pokemon Red/Blue/Yellow, Pokemon Gold/Silver (no RTC)
Game Boy Color (GBC) games are not currently supported. DMG games that are GBC-enhanced will run in DMG mode.

Troubleshooting

No sound

  • Click anywhere on the page first — browsers require a user interaction before playing audio
  • Check that the volume slider isn't at 0
  • Make sure the Mute button isn't active
  • Check that at least some channels are enabled

Game doesn't start or shows glitches

  • Make sure you're loading a .gb file (not .gbc)
  • The game might use an unsupported mapper (MBC2, MBC5, etc.)
  • Try a different ROM dump — corrupted files won't work

Save not loading

  • Battery saves are keyed by ROM title — the exact same ROM must be loaded
  • Save states are also title-matched for safety
  • Check that your browser isn't in private/incognito mode (localStorage may be disabled)

Gamepad not detected

  • Press a button on the controller before opening the remap dialog
  • Some browsers require HTTPS for gamepad access

Technical Details

RUGB is written in Rust, compiled to WebAssembly, and runs entirely client-side. No data is sent to any server — your ROMs and saves stay in your browser.

  • CPU: SM83 — all 512 opcodes implemented
  • Video: Scanline-accurate PPU at 59.73 FPS
  • Audio: Sample-accurate APU at 48 kHz with hardware-matching high-pass filter
  • Source code: github.com/Baptiste-lg/RUGB
